# Metodologias ágeis aplicadas a projetos de Sistemas Embarcados
*06 de maio de 2019*
*Por Poliana de Moraes*

Atualmente utiliza-se o termo VUCA para descrever as intensas e rápidas transformações as quais o mundo atravessa. Este acrônimo do inglês descreve quatro características que vivenciamos diariamente:  

+ Volatilidade (Volatility): as mudanças ocorrem em grande quantidade e cada vez mais rápidas. 
+ Incerteza (Uncertainty): dificuldade de prever resultados futuros, pois os cenários levantados alteram-se rapidamente.
+ Complexidade (Complexity): interdependência entre vários elementos em um mesmo contexto ampliando a dificuldade em prever os resultados futuros.
+ Ambiguidade (Ambiguity): complexidade de análises de cenários diante das diversas alternativas para resolução de uma mesma questão.

Dentro deste panorama, foi desenvolvido a metodologia ágil para gerenciamento de projetos como uma alternativa à gestão tradicional. A metodologia ágil baseia-se em múltiplas iterações curtas na qual cada ciclo estabelece entregas com escopo bem definido, auxiliando a organização a enfrentar o ambiente de incertezas e trazendo a capacidade de se adaptar rapidamente as demandas do mercado.

![](figuras/metodo-agil.avif)

O desenvolvimento de sistemas embarcados pressupõe projetos de hardware e de software. As metodologias ágeis surgiram inicialmente para desenvolvimento de software e nesta área encontra-se bem desenvolvida, porém é possível aplicá-la em todo tipo de projeto. Atualmente observa-se uma tímida aplicação de metodologias ágeis a projetos de hardware. Não é esperado que se aplique uma metodologia ágil a um projeto de hardware exatamente como ela é aplicada ao software, pois são projetos com características distintas.  Uma abordagem  interessante para se iniciar a implantação da metodologia ágil a projetos de hardware é a seleção de melhores práticas e ferramentas ágeis para utilização durante o processo de desenvolvimento buscando como objetivo trazer a equipe a capacidade de adaptação rápida as demandas do mercado.

Como exemplo, uma boa prática para o desenvolvimento de hardware é subdividir o projeto em etapas, na qual cada uma seria tomada como um ciclo do desenvolvimento ágil e no final de cada ciclo obter uma versão demo do produto. Com estas versões demo do produto é possível compartilhar resultados com o cliente e obter um retorno, o qual será incluído no próximo ciclo. Como o projeto é planejado e analisado por etapas, torna-se muito mais rápido adaptar-se a mudanças, além de minimizar os custos com os gastos de modificação do projeto.

Uma outra ferramenta que auxilia a trazer a mentalidade ágil ao projeto são as reuniões diárias, pois elas mantêm a equipe focada no objetivo do ciclo para garantir a entrega. Elas são realizadas todos os dias logo no início do período de trabalho e têm duração de no máximo 15 minutos. Cada membro deve responder a 3 questões:

+ O que eu fiz desde a última reunião?
+ O que eu vou fazer até a próxima reunião?
+ O que me impede de atingir meu objetivo?

Desta forma, espera-se que a equipe possua informação suficiente para melhor se organizar e trabalhar de forma colaborativa. 

Inúmeras são as ferramentas e melhores práticas ágeis que podem ser aplicadas ao desenvolvimento de hardware, ou qualquer outro projeto diferente de software. Vale ressaltar que não existe metodologia ou ferramenta melhor do que a outra. Estas escolhas devem sempre ser pautadas nos objetivos e necessidades do projeto, além de alinhar-se com os valores e contexto da organização.
